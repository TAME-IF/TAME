CallFromMaybe ( ELEMENT, NAME );

Calls a procedure block from an arbitrary element context, but unlike CallFrom, this will not throw an error if it does not exist.

Parameters

ELEMENT Any Element (world, player, object, container, room) The origin element context to call a procedure from (can also be player or room, for current player or room, or world for the world element).
NAME Value or Expression The name of the procedure to call.

Returns

Returns nothing. Cannot be used in expressions.

What It Does

Calls a procedure with the same name as the value parameter in the same element as the provided element (if it exists), or within that element's lineage (parent elements).

Like the name of the procedure, the value can be any type. The procedure called must have a name strictly equal to the value.

Example

See Also

CallMaybe — Similar to CallFromMaybe, but from the current context.
CallFrom — Same as CallFromMaybe, but adds an ERROR cue if it can't find the procedure.

Technical Junk

CallFromMaybe always searches backwards from the specified context despite a block existing either earlier or later in the element's lineage. It also uses strict equality to search for the matching procedure. It must be the same type (e.g. "3", 3.0, and 3 are all NOT equal). So, whatever works for the === operator.
