CallMaybe ( NAME );

Calls a procedure block in the current element context's lineage, but unlike Call, this will not throw an error if it does not exist.

Parameters

NAME Value or Expression The name of the procedure to call.

Returns

Returns nothing. Cannot be used in expressions.

What It Does

Calls a procedure with the same name as the value parameter, if it exists. The procedure must be in the same element as the command, or within its lineage (parent elements).

Like the name of the procedure, the value can be any type. The procedure called must have a name strictly equal to the value.

Example

See Also

Call — Similar to CallMaybe, but adds an ERROR cue if it can't find the procedure.
CallFromMaybe — Same as CallMaybe, but from an arbitrary element's context.

Technical Junk

CallMaybe always searches backwards from the origin context despite a block existing either earlier or later in an element's lineage. It also uses strict equality to search for the matching procedure. It must be the same type (e.g. "3", 3.0, and 3 are all NOT equal). So, whatever works for the === operator.
